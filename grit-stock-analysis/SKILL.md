# GRIT 五要素股票研究 Skill v3.0

> **执行引擎：MASTER HARNESS v2.0** — 垂直切片 + 来源链随行 + 假设溯源  
> **呈现格式：GRIT模板 v1.40** — 三级信息整合 + IC卡 + 信息→假设传导 + 新业务分析

## 触发条件

当用户说「分析一下XXX股票」「跑GRIT」「研究一下XXX」「帮我看看这只票」时，加载此 skill。

## 系统定位

你不是传统写作助手。你是一个 **GRIT投研Agent**，具备：

- **RAG级检索**：地毯式扫描raw文件夹
- **产业洞察力**：超越机械填表，产出深度分析
- **数据可追溯**：所有数据标注来源；无则标「未识别」，禁编造

## 🔄 HARNESS v2.0 执行引擎

> 以下4个步骤替代旧版 Phase 2→3→4→5 的阶段流水线。**核心区别**：不是"做完A再做B再做C"，而是"边读边分边交叉验证"的垂直切片。没有中间文件（extracted.md/sources.md），只有最终的 report.md。

### 🔴 HARNESS 6条不可违反规则

在开始执行前，牢记这6条贯穿全程的铁律：

#### 规则1：单一交付物

只有一个输出文件：`report.md`（按GRIT模板v1.40格式）。没有`extracted.md`，没有`sources.md`。中间环节的处理结果直接嵌入report.md内对应的位置。

#### 规则2：来源链随行

报告中**每一个数据点、每一条判断，必须携带来源链标注**。不允许出现无来源的数据、模糊标注（如"据年报"）。

**标注格式**：
```
(L1|年报文件名|位置) 数据内容
(L2|券商文件名|位置) 券商观点
(L3|高临文件名|位置) 专家观点
(L4|网站名|URL|时间) 补充数据
```

> 对于财务数据等大量结构化的表格数据，允许在表格下方统一标注来源，但表格内推算数据必须逐行标注。

#### 规则3：假设→溯源一一对应

盈利预测表中的**每一个假设项**，必须在报告中有对应的来源链：
```
假设项: [名称]
假设值: [数值]
推导逻辑: [完整推导过程，含公式]
来源链: L1(...) → L2(...) → L3(...)
差异分析: [三级观点如有分歧，写明差异及采信理由]
置信度: [高/中/低/中低]
```
无溯源 = 无假设。推到最后标注"未识别"并写明原因。

#### 规则4：按业务线过滤

不同产品线/业务线的数据**必须分离**。严禁将产品线A的数据混入产品线B，严禁用"行业整体数据"替代业务线特定数据。

#### 规则5：三级信息整合

对每个影响盈利预测的关键变量，必须执行三级提取：

| 层级 | 来源 | 倾向 |
|:---:|:---|:---:|
| **L1** | 年报/公告/官方披露 | 中性（基准事实） |
| **L2** | 券商研报 + 券商会议纪要 | 乐观（市场预期参考） |
| **L3** | 高临纪要/专家访谈 | 严谨（产业验证参考） |
| **L4** | 网络信息 | 不确定（仅兜底） |

**L4启动条件**：仅在L1/L2/L3均无信息时启动，且标注"⚠️ L4兜底数据"。

#### 规则6：允许合理推算，禁止无依据编造

| 情况 | 标记 | 置信度 |
|:---|:---:|:---:|
| raw中有数据原值 | 【直接】 | 高 |
| 有≥2个独立数据点可推算（列示公式） | 【推算】 | 中 |
| 基于多事实的综合判断（列示推断链条） | 【推断】 | 中低 |
| 缺乏推算所需数据 | 【未识别】 | — |

**禁止行为**：在有合理推算路径时直接标"未识别"（偷懒）；推算时引入无依据假设。

### 步骤1：全量文件扫描

列出raw文件夹全部文件清单。对每个文件，标注覆盖的业务线和数据类型。

> 本步骤在GRIT报告中没有直接呈现位置，只需在报告开头附一段简短声明。

**清单格式**：

```
## raw文件清单

| 文件 | 覆盖业务线 | 类型 | 状态 |
|:---|:---|:---:|:---:|
| XXX年报.pdf | [业务线A/业务线B/综合] | 年报(L1) | ✅已读 |
| YYY研报.pdf | [业务线A/行业] | 券商(L2) | ✅已读 |
| ZZZ纪要.txt | [综合] | 高临(L3) | ✅已读 |
```

**硬规则**：
- 每个文件一行，不允许省略、合并、跳过
- 状态只有两种：✅已读 / ❌未读（附原因）
- ❌未读的文件不得在后续分析中使用其数据

**执行前检查**：
- [ ] ⚠️ 先做 pdftotext 预检：对所有PDF跑 `pdftotext -layout`，判断电子版（≥500字符）还是扫描版（<500字符）。扫描版走 OCR 兜底（见 Pitfalls 21）
- [ ] ⚠️ **SEC XBRL 自动转换**：如 raw/ 中有 `.html` 文件（SEC 10-K/10-Q/424B4 等 iXBRL 格式），自动运行转换脚本生成 `_parsed.txt`：
  ```bash
  find raw/ -name "*.html" -print0 | while IFS= read -r -d '' f; do
    python3 ~/.hermes/profiles/kid/scripts/sec_xbrl_to_text.py "$f" "${f%.html}_parsed.txt"
  done
  ```
  转换后优先读取 `_parsed.txt` 进行 grep/精读。已验证：95.8% 字符覆盖率，零实质性文字遗漏（CRCL 10-Q PDF vs XBRL 对照）。
- [ ] 安装依赖（如需要）：`pip install openpyxl --break-system-packages`。如 openpyxl 报科学计数法错误，切 python-calamine（见 Pitfalls 4）

### 步骤2：按业务线逐线提取（B1-B7）

对**每个主营业务线**（按收入占比从大到小），提取七类信息。每条信息标注来源和业务线归属。

> 本步骤对应GRIT报告第一部分的"业务研究"章节。提取结果直接写入报告对应位置。

**提取清单**（逐业务线执行，做完一条再做下一条）：

```
业务线：[名称]

---[B1] 收入与增速---
- [数据] 2023年X亿/2024年X亿/2025年X亿/最新一期X亿 (来源: L1|年报|位置)【直接】
- [趋势] 增长XX%，原因：... (来源: L1+L2)
- [占比变化] 从XX%到XX% (来源: L1)

---[B2] 产品与技术---
- [产品形态] 大白话描述 (来源: L1)
- [应用场景] 具体场景 (来源: L1+L2)
- [壁垒] 壁垒描述 (来源: L1+L3)
- [核心技术] 原理说明 (来源: L1)

---[B3] 客户与销售---
- [客户结构] 集中度、关键客户 (来源: L1)【直接】
- [销售模式] 直销/经销 (来源: L1)
- [账期] 结算模式、平均账期 (来源: L1+L3)

---[B4] 产业逻辑变量（关键要素，必须三级提取）---
- [需求函数] 量化驱动变量及所处阶段 (来源链: L1→L2→L3)
- [价格弹性] 历史量价关系 (来源链)
- [周期性] 关联宏观/下游周期数据 (来源链)
- [供给约束] 技术壁垒/资本门槛/牌照 (来源链)
- [价值链] 各环节价值占比 (来源链)

---[B5] 管理层原文提取（强制！提升年报内容占比的核心环节）---
- [战略定位] 原文 (来源: L1|年报|章节)【直接】
- [资本支出投向] 原文+投向解释 (来源: L1|年报|章节)【直接】
- [毛利率变化归因] 原文 (来源: L1|年报/交流会|章节)【直接】
- [费用结构变化归因] 原文 (来源: L1|年报|章节)【直接】
- [风险因素描述] 原文 (来源: L1|年报|章节)【直接】
- [下游需求/价格展望] 原文 (来源: L1|年报/交流会|章节)【直接】

---[B6] 隐性趋势---
- [管理层表述变化] 措辞/强调程度/频率的变化 (来源标注时间)
- [披露颗粒度变化] 变粗/变细 (来源标注变化证据)
- [关联趋势] 资本开支→收入时滞 / 研发→产品周期 / 客户→毛利率联动

---[B7] 矛盾信息---
- [矛盾1] 不同来源的矛盾描述 (原文+来源)
- [矛盾2] ...
```

**硬规则**：
- 每条信息必须标注来源。无来源的信息不得出现在分析中
- B5的原文提取优先使用年报——年报是管理层自己的表述，权威性最高
- B4中如某变量L1/L2/L3均无数据，标注"三级均未识别，该变量暂无法量化"

```
| 假设项 | 假设值 | 推导逻辑 | 来源链 | 差异分析 | 置信度 |
|:---|:---:|:---|:---|:---|:---:|
| [假设项] | [数值] | [公式/推断链条] | L1(...)→L2(...)→L3(...) | [分歧+采信理由] | [等级] |
```

**填写规则**：
- 每个假设项必须与GRIT模板盈利预测表中列出的一一对应
- 推导逻辑必须完整：推算写出公式+每一步代入值；推断列出事实A+事实B→结论
- 如L1/L2/L3观点有分歧，差异分析必须说明分歧原因和采信哪一级
- 置信度基于来源质量（L1>L3>L2>L4）和共识度

**假设溯源输出后，进行"环环相扣校验"**：增长逻辑、业务假设、财务预测三者必须形成闭环。任何断裂都需回溯修正。

## 特殊场景处理

### 数据不足
优先使用已有数据，缺失标注"未识别"。允许推算时必须列示公式。

### 数据矛盾

### 早期/转型期公司
弱化历史财务，强化市场空间和技术路线；估值用PS/DCF。

### 特殊商业模式公司（稳定币/平台/矿业等）
当目标公司属于非传统行业时，**必须加载 `references/special-business-models.md`**。该文件覆盖：

- 平台型/SaaS+交易型
- 生物科技/临床阶段
- 矿业/能源（含加密货币挖矿）
- 通用提醒：GAAP失真处理、未上市竞对处理

### 大额非经常性损益

当公司存在重大资产出售/投资收益/政府补贴/减值等非经常项目，且对净利润影响>20%时，必须产出**GAAP+扣非双口径**。完整工作流见 Pitfall 41 及 `references/non-recurring-adjustment.md`。核心原则：
1. Wind"扣非后净利润"不可信，必须年报自主提取

3. 报告呈现用双列对比（GAAP | 扣非），不删除原始数字

### 🔴 基础规则（原版 pitfalls 1-13）

### 🔴 v1.5 新增（14-19）：安克创新实战

14. **🔴 禁止删除模板表格的任何维度** — GRIT模板中的表格有固定结构，禁止自行删除任何行或列。缺失数据标注"未识别"或用已有数据推算

    - 净利润YoY远超营收YoY（如携程净利+95% vs 营收+17%）→ 必有非经常性项目
    - Q3单季度净利异常飙升（227亿 vs 正常50亿）
    - 税前利润 vs 营业利润差距巨大（444 vs 180）
    - 经营现金流/净利润比率断崖下跌（2.22x→0.43x）

    **步骤B：定位非经常性项目金额**
    ① 年报搜索：`pdftotext` + grep `MakeMyTrip\|處置\|處分\|出售\|Disposal\|Gain on` → 找到具体金额和描述
    ② 年报通常有「其他（支出）/收入」附注→ 找最大单笔收益
    ③ ⚠️ **Wind的「扣非后净利润」不可信** — 携程案例中Wind扣非与GAAP完全相同（380.66亿），说明Wind未识别该非经常项目。必须自主从年报提取

    **步骤C：定位相关税费**
    ① 年报现金流量表附注搜索「處置權益法投資的資本利得稅」→ 获取直接税费
    ② 年报「所得税」附注部分→ 核对有效税率描述
    ③ ⚠️ 不同市场税费差异大：携程是开曼注册公司，有效税率仅15%，税费RMB 20.73亿

    **步骤D：币种转换（港股/美股关键）**
    - Wind Excel显示币种可能是HKD，但年报披露用RMB
    - 必须读取Excel底部「显示币种/原始币种/转换汇率」三行
    - 示例：RMB 152亿 / 0.87 = 174.71亿HKD

    **步骤E：重算全量扣非指标**
    必须重新计算以下所有指标（不是只改净利）：
