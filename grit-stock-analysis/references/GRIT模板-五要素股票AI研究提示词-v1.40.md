更新时间：2026.05.08

> **本版本核心更新**：引入"三级信息整合框架"（L1年报→L2券商→L3高临→L4网络兜底），对每个影响盈利预测的关键要素执行三级提取、差异分析、综合判断，并通过"三级信息整合卡（IC）"和"信息→假设传导表"实现从原始信息到盈利预测假设的显式逻辑链条。详见产业分析中的【三级信息速览】、盈利预测中的【信息→假设传导表】和【预测假设与可靠性说明】。

## 【全覆盖6条规则（不可违反）】

> 以下6条规则贯穿整个报告的生成过程，由HARNESS v2.0执行框架强制执行。

### 规则①：来源链随行

报告中**每一个数据点、每一条判断**必须携带来源链标注。

**标注格式**：
```
(L1|年报文件名|位置) 数据内容
(L2|券商文件名|位置) 券商观点
(L3|高临文件名|位置) 专家观点
(L4|网站名|URL|时间) 补充数据
```

> 对于大量结构化的财务数据表格，允许在表格下方统一标注主数据来源（如"本表数据全部来源于XXX.xlsx/YYY年报"），但表格内如有推算数据必须逐行标注。

### 规则②：信息性质标注

每个判断性结论必须标注信息性质：

| 标注 | 含义 | 要求 |
|:---|:---|:---|
| 【直接】 | 原文照抄/数据原值 | 标注来源位置 |
| 【推断】 | 基于多事实的综合判断 | 列示推断链条（事实A[来源]+事实B[来源]→结论）|
| 【推算】 | 基于数学关系的计算 | 列示公式和每一步代入值 |
| 【未识别】 | 无法判断 | 写明原因 |

**组合示例**：
```
(L1|年报全文.md|第34页)【直接】2025年电力行业收入2.85亿元，同比+24.24%
(L1+L3)【推断】行业装机+22.4%×份额稳定→增速约25-28%
(L3→推算)【推算】1台风机服务费2000元×1000台÷2(保守系数)=0.1亿
```

### 规则③：三级信息整合

对每个影响盈利预测的关键变量（收入增速、毛利率、费用率、市场份额等），必须执行：

```
L1(年报) → L2(券商研报+券商会议纪要) → L3(高临纪要/专家访谈) → L4(网络，仅兜底)
```

标注格式：
```
L1(年报|位置): xxx
L2(券商|位置): xxx
L3(高临|位置): xxx
差异分析: xxx
采信: xxx (理由: xxx)
```

### 规则④：按业务线过滤

不同产品线/业务线的数据**必须分离分析**。禁止将业务线A的行业数据混入业务线B的描述中，禁止用"行业整体数据"替代"该业务线特定数据"而不加说明。

各业务线的分析段落中只引用已归属到该业务线的素材。

### 规则⑤：允许合理推算，禁止无依据编造

有≥2个独立数据点可推算时→【推算】列示公式。完全无数据时才标【未识别】。

### 规则⑥：假设→溯源一一对应

盈利预测表中每个假设项必须在报告内有对应的来源链。无溯源 = 无假设。推到某一步无法溯源时标注"未识别"。

4. 关于"1变2"结论，可陈述"判定结论为：完全符合/部分符合/不符合'未来3年业绩1变2'"，但不得展开交易意见（交易意见仅在第三部分综合结论中展开）

## 第一部分 公司业务与基础信息

> **⚠️ AI工作要求（不可跳过）**：
> 1. **第一部分是整个报告的基础主体，不是可选项**。agent禁止因"内容量大"或"先写五要素再说"而跳过第一部分。
> 2. **第二部分的五要素分析100%依赖第一部分的产出**：需求要素中的行业规模与增速需要第一部分的业务定位；竞争要素中的对手对比需要第一部分的客户分析和产业逻辑；资本要素中的投入产出效率需要第一部分的财务视角；估值要素中的盈利预测需要第一部分的业务增速假设。
> 3. **没有第一部分，第二部分就是无源之水**。如agent试图跳过第一部分，必须被强制拉回。
> 4. **输出顺序强制**：必须先完成第一部分全部内容 → 填写跨引用索引 → 再进入第二部分。

##### 【产业分析】

> **⚠️ 分析纪律**：本部分不是独立的行业研究报告，其唯一目的是**为盈利预测提供假设依据**。每个分析bullet必须思考："这个判断对收入/成本/费用预测意味着什么？" 禁止因raw资料中宏观行业内容丰富而展开泛泛描述——如果某个宏观趋势与公司具体业务线无直接传导关系，简要带过即可，不要被带偏。

> **信息性质标注规范（强制）**：本部分所有判断性结论必须同时标注【信息层级】和【信息性质】。
> - 【信息层级】：【L1-年报】【L2-券商】【L3-高临】【L4-网络】——信息来自哪类原始资料
> - 【信息性质】：【直接】原文照抄 → 【推断】基于多事实综合判断（必须列示推断链条） → 【推算】基于数学关系计算（必须列示公式） → 【未识别】无法判断
> - 组合示例：【L1-年报+高临】【推断】表示"综合年报事实和高临观点做出的合理推断"
>
> 全业务三级信息整合速览见"一、公司业务"末尾独立章节。

**需求侧分析**（每bullet需有数据或raw原文支撑，无则标注"信息不足"）：

- **需求函数**：列出至少2个可量化驱动变量，说明各变量当前所处阶段（上升/下降/拐点）；如Q = f(下游资本开支, 替代技术价格, 政策补贴)
- **价格弹性**：基于历史价格变动与销量关系或客户结构特征给出判断，避免主观断言
- **周期性判断**：关联具体宏观经济指标或下游行业周期数据，给出强/弱/抗周期判定
- **采购决策特征**：下游客户的决策流程、周期、转换成本（需有客户访谈或订单数据支撑）

**供给侧分析**（每bullet需有数据或raw原文支撑，无则标注"信息不足"）：

- **供给约束**：技术壁垒高度、资本门槛金额、牌照稀缺性（定量描述，如"全国仅XX张牌照"）
- **产能周期**：从资本开支决策到投产的具体时长（月/年），基于公司历史扩产数据或行业平均
- **成本结构**：固定成本vs可变成本比例（基于财务数据计算），规模经济效应是否有数据支撑

**价值链分配**（尽量量化，无法量化时说明原因）：

- 产业链各环节的价值分配比例（估算百分比，标注估算依据）
- 公司所处环节的价值捕获能力（高/中/低，附判断依据）
- 上下游议价能力对比（引用账期、集中度、转换成本等具体证据）

**隐性信息解析**：

> 基于raw资料中该业务相关的披露信息识别信号。**有则引用原文（标注时间/场合），无则逐项明确标注"未识别到明显变化"，禁止编造。**

- **管理层表述变化**：对业务前景的措辞是否变化？对风险因素的强调程度是否变化？提及频率是否变化？
  - *填写规范*：引用管理层原文（标注时间/场合），如"2024年报表述为'快速增长'，2025半年报调整为'稳健发展'"；如无变化证据，写"未识别到管理层表述的显著变化"
- **数据颗粒度变化**：该业务的披露颗粒度是否变粗或变细？是否有新增或删除的披露项？
  - *填写规范*：引用具体披露变化，如"2024年起不再披露该业务的分地区收入，颗粒度变粗"；如无变化，写"未识别到披露颗粒度的显著变化"
- **关联趋势**：资本开支与收入增长的时滞关系？研发投入与产品推出的周期关系？客户结构与毛利率的联动？
  - *填写规范*：用数据说明时滞/周期长度，如"资本开支峰值在2023Q2，收入增长在2024Q1，时滞约3个季度"；如无数据，写"未识别到显著的关联趋势"
- **矛盾信息调和**：如遇到矛盾信息，优先引用【三级信息速览】中的差异分析和综合判断；如速览表未覆盖的矛盾，列出矛盾双方、分析立场与证据权重、给出最可能判断及置信度
  - *填写规范*：引用矛盾双方原文，标注来源；如无矛盾信息，写"未识别到关于该业务的显著矛盾信息"

#### 4、重大新业务分析

> **本节定位**：针对目标公司正在酝酿、当前收入占比极低（接近为零），但未来1-2年增量贡献重大且确定性高的新业务。与三大主营业务分析不同，本节**不依赖历史财务数据**，而是聚焦"确定性验证+增量贡献测算"。
> 
> **核心原则**：不是"炒概念"，而是验证"火车是否已经启动"。
